## 블루프린트 기반 추가 구현 항목

아키텍처 블루프린트에 명시되었으나 아직 구현되지 않은 항목들입니다.

### 성능 최적화 (Performance Optimization)

- **코드 분할 (Code Splitting):**
  - **내용:** `React.lazy`와 `Suspense`를 사용하여 `ChatView`, `HistoryView`, `SettingsView` 등 각 뷰를 동적으로 임포트하여 초기 로딩 시간을 단축합니다.

- **가상 스크롤 (Virtual Scrolling):**
  - **내용:** 채팅 메시지나 히스토리 목록이 매우 길어질 경우, `react-window`와 같은 라이브러리를 사용하여 화면에 보이는 부분만 렌더링함으로써 DOM 요소의 수를 최소화하고 렌더링 성능을 향상시킵니다.

### 배포 전략 (Deployment Strategy)

- **CI/CD 파이프라인 구축:**
  - **내용:** GitHub Actions를 사용하여 `v*` 태그가 푸시될 때마다 자동으로 테스트, 빌드, 패키징 및 GitHub 릴리즈 생성을 수행하는 CI/CD 파이프라인을 구축합니다.

### 테스트 전략 (Testing Strategy)

- **E2E(End-to-End) 테스트 자동화:**
  - **내용:** Playwright를 사용하여 실제 브라우저 환경에서 확장 프로그램을 로드하고, 사이드바 열기, 메시지 전송, 컨텍스트 메뉴 사용 등 핵심 사용자 시나리오를 자동으로 테스트하는 E2E 테스트 스위트를 구축합니다.

## Gemini 디자인 시스템 기반 UI/UX 개선 제안

기능을 넘어 '아름다움'과 '즐거움'을 주는 디자인을 목표로, Google Gemini의 시각적 특징을 기반으로 한 개선 제안입니다.

### 1. 타이포그래피 및 레이아웃 (Typography & Layout)

- **전용 서체 도입 및 타이포그래피 시스템 정립:**
  - **문제점:** 현재 시스템 기본 폰트를 사용하여 개성이 부족합니다.
  - **개선 제안:** Google Fonts의 `Inter` 또는 `Figtree` 같이 Gemini와 유사한 느낌을 주는 현대적인 산세리프 폰트를 도입합니다. `@fontsource` 패키지를 사용하여 폰트를 프로젝트에 포함시키고, 제목, 본문, 캡션 등 위계에 따른 명확한 타이포그래피 시스템(크기, 굵기, 자간)을 정의하여 전반적인 가독성과 심미성을 향상시킵니다.

- **8px 그리드 시스템 기반의 정교한 여백 관리:**
  - **문제점:** 컴포넌트 간 간격이 일관되지 않아 다소 정돈되지 않은 느낌을 줍니다.
  - **개선 제안:** 모든 UI 요소의 마진, 패딩을 4px 또는 8px 단위의 그리드 시스템에 맞춰 재조정합니다. 이를 통해 시각적 리듬감을 형성하고, 사용자가 무의식적으로 안정감과 정돈된 느낌을 받게 합니다.

### 2. 색상 및 비주얼 이펙트 (Color & Visual Effects)

- **오로라 그라데이션 배경 (Aurora Gradient Background):**
  - **문제점:** 단색 배경은 다소 단조롭습니다.
  - **개선 제안:** 다크 모드에서 Gemini 특유의 '오로라' 효과처럼, 여러 색상이 미묘하게 섞여 천천히 움직이는 동적인 그라데이션 배경을 채팅 뷰에 적용합니다. 이는 시각적 즐거움을 주고, 앱에 생명력을 불어넣습니다.

- **세련된 액센트 컬러 팔레트 도입:**
  - **문제점:** 현재의 파란색 액센트 컬러가 다소 표준적입니다.
  - **개선 제안:** Gemini에서 사용하는 것처럼, 더 부드럽고 채도가 높은 파란색, 보라색, 주황색 계열의 액센트 컬러 팔레트를 새롭게 정의합니다. 버튼, 링크, 하이라이트 등 상호작용 요소에 이 색상들을 적용하여 더 활기차고 현대적인 인상을 줍니다.

- **글래스모피즘 (Glassmorphism) 효과 적용:**
  - **문제점:** UI 요소들이 모두 불투명하여 깊이감이 부족합니다.
  - **개선 제안:** 컨텍스트 메뉴 팝오버나 사이드바 헤더에 `backdrop-filter: blur()` 속성을 사용하여 뒷배경이 은은하게 비치는 '프로스티드 글래스(Frosted Glass)' 효과를 적용합니다. 이를 통해 UI의 깊이감을 더하고 고급스러운 느낌을 연출합니다.

### 3. 모션 및 마이크로 인터랙션 (Motion & Micro-interactions)

- **순차적 메시지 애니메이션 (Staggered Animation):**
  - **문제점:** 새 대화 로드 시 모든 메시지가 한 번에 나타나 밋밋합니다.
  - **개선 제안:** 대화 기록을 불러올 때, 각 메시지 버블이 0.05초 정도의 미세한 시간차를 두고 순차적으로(staggered) 나타나도록 애니메이션을 적용합니다. 이는 사용자의 시선 흐름을 자연스럽게 유도하고, UI가 더 동적으로 느껴지게 합니다.

- **의미있는 아이콘 트랜지션 (Meaningful Icon Transitions):**
  - **문제점:** 아이콘의 상태 변화가 즉각적이어서 인지하기 어렵습니다.
  - **개선 제안:** '복사' 아이콘을 클릭하면 부드럽게 '체크' 아이콘으로 변형(morph)되는 애니메이션을 추가합니다. '전송' 버튼 또한 클릭 시 '정지' 아이콘으로 자연스럽게 전환되도록 하여, 상태 변화를 명확하고 아름답게 전달합니다.

- **Gemini 스타일 로딩 인디케이터:**
  - **문제점:** 현재의 스피너 로딩은 일반적입니다.
  - **개선 제안:** Gemini에서 사용하는 것처럼, 반짝이는 빛이 좌우로 움직이는 'Shimmer' 효과나, 크기가 다른 세 개의 점이 파동처럼 움직이는 로딩 인디케이터로 교체하여 AI가 '생각'하고 있다는 느낌을 세련되게 전달합니다.


### 4. 아이코노그래피 및 시각 요소 (Iconography & Visual Elements)

- **사용자 및 AI 아바타 디자인:**
  - **문제점:** 사용자(User)와 AI(Assistant)의 구분이 텍스트로만 이루어집니다.
  - **개선 제안:** 사용자에게는 이니셜 기반의 단색 아바타를, AI에게는 Gemini의 별 모양 로고를 단순화한 세련된 아바타를 각 메시지 옆에 표시합니다. 이는 대화의 주체를 시각적으로 즉시 구분하게 해주고, 브랜드 아이덴티티를 강화합니다.

## UI/UX 전문가 제안 개선사항 (Expert UI/UX Suggestions)

기능적으로 완성된 현재 버전을 넘어, 사용자의 작업 흐름을 가속화하고 더 직관적인 경험을 제공하기 위한 전문가 제안입니다.

### 1. 채팅 경험 고도화 (Chat View)

- **실시간 타이핑 효과 (Live Typing Effect):**
  - **문제점:** 현재는 AI 응답 청크가 도착하는 대로 텍스트가 붙여넣기 되어 다소 기계적으로 보입니다.
  - **개선 제안:** 실제 사람이 타이핑하는 것처럼 한 글자씩 나타나는 타이핑 커서 효과를 적용하여, 응답이 생성되고 있다는 느낌을 시각적으로 전달하고 더 생동감 있는 경험을 제공합니다.

- **페이지 컨텍스트 인용 및 하이라이팅 (Source Citation & Highlighting):**
  - **문제점:** 'Chat with Page' 기능으로 답변을 받아도, AI가 페이지의 어느 부분을 참고했는지 알 수 없어 신뢰도가 떨어지고 사용자가 직접 내용을 다시 찾아야 합니다.
  - **개선 제안:** AI 답변 중 페이지 콘텐츠를 인용한 부분에 각주(e.g., `[1]`, `[2]`)를 표시합니다. 이 각주에 마우스를 올리거나 클릭하면, 원본 웹 페이지의 해당 텍스트 주변이 하이라이트되거나 팝업으로 표시되어 정보의 출처를 즉시 확인할 수 있게 합니다.

- **후속 질문 제안 (Suggested Follow-up Questions):**
  - **문제점:** 사용자는 다음에 무엇을 질문해야 할지 막막할 수 있습니다.
  - **개선 제안:** AI의 답변이 완료된 후, 해당 답변과 관련된 2~3개의 예상 후속 질문을 버튼 형태로 제공합니다. 사용자는 버튼 클릭만으로 자연스럽게 대화를 이어나갈 수 있습니다.

- **대화 자동 요약 (Conversation Summary):**
  - **문제점:** 대화가 길어지면 이전 내용을 파악하기 어렵습니다.
  - **개선 제안:** 10회 이상 대화가 오가면, 자동으로 대화의 핵심 내용을 요약한 문장을 생성하여 채팅 상단에 고정합니다. (예: "요약: React 성능 최적화 방법에 대해 논의 중")

### 2. 기록 관리 편의성 향상 (History View)

- **전체 기록 검색 (Full-text Search):**
  - **문제점:** 현재는 원하는 과거 대화를 찾으려면 수동으로 스크롤해야 합니다.
  - **개선 제안:** History View 상단에 검색창을 추가하여, 모든 대화 내용에서 키워드 검색이 가능하도록 합니다. 검색 결과는 키워드가 포함된 메시지와 해당 대화 목록을 보여줍니다.

- **날짜별 그룹화 (Group by Date):**
  - **문제점:** 모든 기록이 단순 목록으로 나열되어 시간적 맥락을 파악하기 어렵습니다.
  - **개선 제안:** '오늘', '어제', '지난 7일', '지난 30일' 등 시간 순서에 따라 대화 목록을 자동으로 그룹화하여 보여줍니다.

- **세션 즐겨찾기 및 다중 선택 (Pin & Bulk Actions):**
  - **문제점:** 중요한 대화를 다시 찾기 어렵고, 여러 대화를 한 번에 관리할 수 없습니다.
  - **개선 제안:** 각 세션에 '즐겨찾기(Pin)' 아이콘을 추가하여 상단에 고정하는 기능을 제공합니다. 또한, 편집 모드를 두어 여러 세션을 체크박스로 선택한 후 한 번에 삭제하는 기능을 추가합니다.

### 3. 컨텍스트 메뉴 기능 강화 (Popover)

- **선택 텍스트 즉시 교체 (Direct Replacement):**
  - **문제점:** 현재는 팝오버에서 결과를 '복사'한 후, 사용자가 직접 원래 위치에 '붙여넣기' 해야 하는 번거로움이 있습니다.
  - **개선 제안:** 팝오버에 '바꾸기(Replace)' 버튼을 추가합니다. 이 버튼을 클릭하면, 웹 페이지에서 선택했던 원본 텍스트가 AI가 제안한 결과물로 즉시 교체됩니다.

- **빠른 추가 수정 (Quick Refinements):**
  - **문제점:** 결과물이 만족스럽지 않을 때, 사이드바로 이동하여 다시 명령해야 합니다.
  - **개선 제안:** 팝오버 내부에 '더 짧게', '더 길게', '더 격식있게', '더 친근하게' 같은 빠른 수정 버튼을 추가하여, 1-click으로 결과물을 추가 가공할 수 있게 합니다.

### 4. 전역적 사용자 경험 (Global UX)

- **커맨드 팔레트 (Command Palette):**
  - **문제점:** 모든 기능을 사용하려면 마우스 클릭에 의존해야 합니다.
  - **개선 제안:** `Ctrl+K` (macOS: `Cmd+K`) 단축키를 누르면, '새 채팅', '페이지와 채팅 켜기/끄기', '설정으로 이동' 등 주요 기능을 검색하고 즉시 실행할 수 있는 커맨드 팔레트를 화면 중앙에 띄웁니다.

- **최초 사용자 온보딩 (First-use Onboarding):**
  - **문제점:** 신규 사용자는 어떤 기능이 있는지 발견하기 어렵습니다.
  - **개선 제안:** 확장 프로그램 설치 후 처음 사이드바를 열었을 때, 주요 기능(사이드바 채팅, 컨텍스트 메뉴, 페이지와 채팅)의 위치와 사용법을 간략하게 안내하는 가이드 투어를 제공합니다.

- **LLM 고급 설정 (Advanced LLM Settings):**
  - **문제점:** 모든 사용자가 동일한 LLM 응답 스타일을 원하지 않습니다.
  - **개선 제안:** 설정 페이지에 '고급 설정' 섹션을 추가하여, `temperature` (창의성), `max_tokens` (최대 길이) 등 LLM 파라미터를 사용자가 직접 조절할 수 있게 합니다. '기본 시스템 프롬프트'를 수정하는 기능도 제공하여 전문가 사용자의 자유도를 높입니다.

## 기존 향후 개선사항 (Post-v1.0)

### High Priority (v1.1 목표)

- **RAG for Chat with Page**: 페이지 컨텍스트 처리 시, 벡터 임베딩 기반의 시맨틱 검색을 도입하여 연관성 높은 부분만 컨텍스트로 사용하여 정확도와 토큰 효율성을 높입니다.
- **음성 입력**: Web Speech API를 연동하여 사용자가 음성으로 메시지를 입력할 수 있는 기능을 추가합니다.
- **사용자 정의 프롬프트 템플릿**: 자주 사용하는 프롬프트(예: 이메일 초안 작성, 코드 리뷰 요청)를 저장하고 불러올 수 있는 템플릿 기능을 추가합니다.
- **키보드 단축키 설정**: 사이드바 열기, 컨텍스트 메뉴 기능 호출 등을 위한 사용자 정의 키보드 단축키를 설정하는 기능을 제공합니다.
- **채팅 내보내기 (Markdown/PDF)**: 개별 채팅 대화 내용을 마크다운 또는 PDF 파일로 내보내는 기능을 추가합니다.

### Medium Priority (v1.2 목표)

- **멀티 LLM 제공자 지원**: OpenAI 외에 Anthropic Claude, Google Gemini 등 다른 LLM 모델을 선택하여 사용할 수 있도록 API 클라이언트를 확장합니다.
- **채팅 기록 검색**: 전체 채팅 기록에서 키워드로 특정 대화나 메시지를 검색하는 기능을 구현합니다.
- **대화 분기 (Conversation Branching)**: 특정 AI 답변 시점부터 다른 질문을 하여 대화를 여러 갈래로 나눌 수 있는 기능을 추가합니다.
- **코드 실행**: LLM이 생성한 코드 스니펫(Python, JavaScript 등)을 샌드박스 환경에서 즉시 실행하고 결과를 확인할 수 있는 기능을 구현합니다.

### Low Priority (v1.3+ 목표)

- **협업 세션**: 채팅 세션을 다른 사용자와 공유하거나, 여러 기기에서 실시간으로 동기화하는 기능을 구현합니다. (클라우드 백엔드 필요)
- **플러그인 시스템**: 외부 개발자나 사용자가 직접 새로운 기능을 추가할 수 있는 플러그인 아키텍처를 설계합니다.
- **오프라인 모드**: 네트워크 연결 없이도 이전에 캐시된 LLM 응답을 조회할 수 있는 제한적인 오프라인 모드를 지원합니다.
